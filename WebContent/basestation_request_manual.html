Base stations Request Manual


The base-station user is special type of user that is different than app users (i.e., customers) and police users. This type of user is authorized to send and receive the information of sensors.


A base station should take the following steps to update the server DB with the information of its sensors:


  1. Authentication
    The base station users also need authentication, but their method of authentication is by using username/password and is different than customers (which use JWToken api).
    This user should first use the signin request to log into the system:
    curl --cookie cookies.txt --cookie-jar cookies.txt -s -i -d "login_type=basestation&username=basestation&password=basestationpassword" http://95.156.255.90:8080/Parking/signin

  2. Selecting the City
    While the city is determined automatically for the app user, the basestation user needs to select the city before sending other requests. Use the following request:
    curl --cookie cookies.txt --cookie-jar cookies.txt -s -i -d "city_id=1" http://95.156.255.90:8080/Parking/selectcity

  3. Updating Sensor Info:
    At this point, the server is ready to accept sensor updates. The following command updates the information of 2 sensors:
    curl -b cookies.txt -s -i -d "id=1254453&full=true&t1=12:34:45&t2=12:59:20&id=1254454&full=false&t1=12:04:45&t2=12:59:21" http://95.156.255.90:8080/Parking/updatesensors

  4. Reading the Status of a Sensor:
    There is also an additional request implemented to enable the test team to fetch what is stored in the DB for a sensor by passing its id. You can use the following command:
    curl -b cookies.txt -s -i -d "id=1254453" http://95.156.255.90:8080/Parking/readsensor

Sensor Info GUI

There is also a GUI implemented for the test team to easily see the information of a sensor that is stored in the database.

This GUI can be accessed through the following link:

http://95.156.255.90:8080/Parking/sensors.jsp

From this page you can sign in (default username and password are ‘basestation’ and ‘basestationpassword’), select the city, and start searching for sensors using their id.
