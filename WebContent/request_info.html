spotinfo

http://domain:port/spotinfo?sector_id=1

{“id”:1, “location_x”:5, “location_y”:5, “park_capacity”:10, “available_park”:8, “price_rates”:[{“id”:1, “description”:”cheap”, “price”:2000}, {“id”:2, “description”:”Morning”, “price”:3000}]}

A sector object that has an array of price_rate objects, describing pricing information.

4

spotinfo

http://domain:port/spotinfo?sector_id=1&segment_id=1

{“id”:1, “location_x”:5, “location_y”:5, “park_capacity”:10, “available_park”:8, “price_rates”:[{“id”:1, “description”:”cheap”, “price”:2000}, {“id”:2, “description”:”Morning”, “price”:3000}], “segment”:{“id”:1, “park_capacity”:5, “available_park”:3}}

This sector object could have two attributes segment or spot if ids are given.

NOTE: the given segment must be under the given sector.

spotinfo

{“id”:1, “location_x”:5, “location_y”:5, “park_capacity”:10, “available_park”:8, “price_rates”:[{“id”:1, “description”:”cheap”, “price”:2000}, {“id”:2, “description”:”Morning”, “price”:3000}], “segment”:{“id”:1, “park_capacity”:5, “available_park”:3}, “spot”:{“id”:8, “base_station_id”:13, “parkometer_id”:12, “sensor_id”:24, “status”:”FULL”}}

This sector object could have two attributes segment or spot if ids are given.



NOTE: the given spot must be under the given segment, same for segment being under sector.

8

calcprice

http://domain:port/calcprice?sector_id=1&segment_id=1&rate_id=1&park_time=4

{“id”:1, “location_x”:5, “location_y”:5, “park_capacity”:10, “available_park”:8, “price”:20000, “segment”:{“id”:1, “park_capacity”:5, “available_park”:3}}

Returns a sector object that includes sector info, segment object, and an attribute called price which gives the calculated price for the given time.

NOTE: park_time is the number of quarters requested, for example 4 is equal to 1 hour.

9

{ “status”:”unsuccessful”, “message”: “sector/segment id must be given.”}

Both sector id and segment id are required for this command.

10

This segment is full and there are no spots to rent.

{ “resource”:{“sector_id”:1, “segment_id”:1, “spot_id”:1}, “payment”:{“status”:”successful”, “transaction_id”:1, “price”:20000} }

A transaction object that has (1) resource object containing the information of the rented parking spot, and (2) payment object containing the results and info of the payment.
